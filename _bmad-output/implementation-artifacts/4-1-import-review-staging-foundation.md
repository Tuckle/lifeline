# Story 4.1: Import Review Staging Foundation

Status: done

## Story

As an authenticated Lifeline user,  
I want imported data to appear in a separate staged review area,  
so that my main life-line is not flooded with raw activity or note records.

## Acceptance Criteria

1. Given I am authenticated, when I open the Imports route, then I see an Import Review surface separate from the main Timeline, and the page explains that imports remain staged until I choose what matters.
2. Given import storage is implemented, when import source and import record tables are created, then they are user-owned through Supabase Auth identity and RLS, and imported records are stored separately from primary timeline events.
3. Given imported records exist, when they are queried for the Imports page, then they include lifecycle state, sync status, source metadata, timestamp/date range, and content summary fields, and they do not automatically appear as primary timeline events.
4. Given there are no connected sources or staged records, when I open Imports, then the empty state explains imports calmly and offers actions to connect RescueTime or import notes.
5. Given import lifecycle states are implemented, when a record is created, then its initial lifecycle state is `staged`, and later states use explicit lifecycle values rather than ad hoc booleans.
6. Given import staging code logs errors, when logs or diagnostics are written, then they capture stable error codes and technical context only and do not store sensitive note content or detailed activity content.
