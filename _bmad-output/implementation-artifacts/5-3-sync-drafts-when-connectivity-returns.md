# Story 5.3: Sync Drafts When Connectivity Returns

Status: done

## Story

As an authenticated Lifeline user,  
I want offline drafts to sync when connectivity returns,  
so that captured memories become part of my private timeline.

## Acceptance Criteria

1. Given I have one or more local-only drafts, when the app detects connectivity has returned, then it offers or begins sync according to the MVP sync behavior, and each draft shows a clear sync status.
2. Given a draft is syncing, when sync is in progress, then the draft state changes to syncing or sync pending, and I can distinguish it from fully synced timeline events.
3. Given a draft sync succeeds, when the server creates the timeline event, then the synced event appears as a normal user-owned timeline event, and the local draft state changes to synced or is safely cleared after confirmation.
4. Given multiple drafts exist, when syncing occurs, then each draft has its own sync result, and one failed draft does not prevent successful drafts from syncing.
5. Given sync succeeds after refresh or route change, when I return to the timeline, then synced events remain visible in the correct timeline position, and duplicate events are avoided where draft identity can be matched.
6. Given sync uses server-side mutation code, when the draft is persisted, then ownership is scoped to the authenticated user, and the mutation returns the shared `ActionResult<T>` shape.
