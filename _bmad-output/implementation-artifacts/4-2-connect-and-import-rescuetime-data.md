# Story 4.2: Connect and Import RescueTime Data

Status: done

## Story

As an authenticated Lifeline user,  
I want to connect RescueTime and import activity context,  
so that I can review what I was doing during selected life periods without polluting my main timeline.

## Acceptance Criteria

1. Given I am on the Imports route, when I choose Connect RescueTime, then I see a clear permission explanation for what Lifeline will access and why, and the copy states that imported data will remain staged until I curate it.
2. Given I authorize RescueTime successfully, when the connection is saved, then an import source record is created for my authenticated account, and any token or secret handling stays server-side.
3. Given I trigger a RescueTime import, when activity data is fetched, then Lifeline normalizes records into staged import records with source metadata, timestamps/date ranges, sync status, and activity summaries, and no RescueTime record becomes a primary timeline event automatically.
4. Given the RescueTime import succeeds, when I return to Import Review, then I can see newly staged RescueTime records grouped by date or period, and each record is labeled as RescueTime-sourced.
5. Given the RescueTime import partially fails, when results are shown, then successfully imported records remain available, and failed records or failed ranges are identified with retry guidance.
6. Given authorization fails, expires, or is canceled, when I return to Lifeline, then I see a calm source-specific error state, and I can retry, reconnect, ignore, or disconnect as appropriate.

## Dev Notes

- Story 4.1 is done and pushed to `origin/main` at commit `b48af17`.
- Official RescueTime docs verified on 2026-05-06:
  - API keys are suitable for personal use.
  - Analytic Data API key endpoint is `https://www.rescuetime.com/anapi/data`.
  - Required parameters include `key` and `format`; JSON returns `row_headers` plus `rows`.
  - Useful query parameters include `perspective=interval`, `restrict_kind=productivity`, `interval=hour`, `restrict_begin`, and `restrict_end`.
- Keep this as a narrow founder-use slice. Do not add OAuth app registration, background workers, broad historical sync, or analytics dashboards.
- Do not expose the RescueTime key to client components or `NEXT_PUBLIC_*` variables.
- Imported records must remain in `import_records` with lifecycle `staged`; never create `timeline_events` from this story.
